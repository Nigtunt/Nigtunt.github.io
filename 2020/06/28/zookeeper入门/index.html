zookeeper介绍

Zookeeper本身是Hadoop生态园的中的一个组件，Zookeeper强大的功能， 在Java分布式架构中，也会频繁的
使用到Zookeeper。

结构类 Unix文件系统路径节点，可以往这个节点存储或获取数据。

zookeeper下载安装

地址： https://downloads.apache.org/zookeeper/zookeeper-3.4.14/

下载解压后进入bin目录，点击zkServer.cmd启动

会闪退，然后编辑zkServer.cmd 增加一个pause

然后可以看到错误信息，发现好像因为没有配置conf

然后去conf目录下添加一个conf就行了

然后使用zkCli.cmd 连接中心，进行一些基本的操作

zookeeper架构

zookeeper架构图

每一个节点都被成为znode

每一个znode中都可以存储数据

节点名称是不允许重复的

znode类型

四种zonde

  1. 持久节点

    永久保存在zookeeper中

  2. 持久有序节点

    永久保存在zookeeper中，他会添加一个有序的序号

  3. 临时节点

    当存储的客户端和Zookeeper服务断开连接时，这个临时节点自动删除

  4. 临时有序节点

    当存储的客户端和Zookeeper服务断开连接时，这个临时节点自动删除，他会给节点添加一个有序的序号。

    /xx -> /0x000001

zonde改变时，会通知监听当前的znode的客户端

zookeeper集群

zookeeper集群架构图

1.主从之分

2.Zookeeper集群中必须有master节点，否则Zookeeper无法正常工作的

3.master:执行读写操作

4.slave:执行读

5.Zookeeper集群的投票， 在没有Master几点时，会重新选举。

zookeeper集群的角色

1、Leader

​ Master主节点
2、Follower（默认从节点）

​ 从节点，参与选举全新的Leader
3、Observer

​ 从节点，不参与投票

4、looking

​ 正在找Leader节点

zookeeper投票策略

1、每一个Zookeeper服务 都会被分配一个全局唯一 的myid, myid是一个数字。

2、Zookeeper在执行写数据时， 每一个节点都有一个自己的FIFO的队列。保证写每一个数据的时候， 顺序是
不会乱的，Zookeeper还会给每一个数据分配一个全局唯一的zxid, 数据越新zxid就越大。

选举Leader:

1.选举出zxid最大的节 点作为Leader。

2.在zxid相同的节点中， 选举出一个myid最大的节点， 作为Leader。

Compose 使用的三个步骤：

  • 使用 Dockerfile 定义应用程序的环境。
  • 使用 docker-compose.yml 定义构成应用程序的服务，这样它们可以在隔离环境中一起运行。
  • 最后，执行 docker-compose up 命令来启动并运行整个应用程序。
